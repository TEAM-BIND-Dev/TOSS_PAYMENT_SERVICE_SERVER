# ADR 003: 토스 페이먼츠 2단계 승인 방식

**날짜**: 2025-11-21  
**상태**: 승인됨  
**결정자**: TeamBind

---

## 결정 사항

**토스 페이먼츠의 2단계 승인 방식을 채택합니다.**

```
1. 클라이언트: Toss SDK로 결제창 호출
2. 토스: successUrl로 리다이렉트 (paymentKey 포함)
3. 백엔드: Toss 승인 API 호출
4. 토스: 최종 승인 결과 반환
```

---

## 근거

### 보안
- 클라이언트에서 직접 결제 완료 불가
- 백엔드 검증 필수 (금액, 상태 등)
- 서버사이드 승인으로 변조 방지

### 토스 공식 플로우
- 토스가 권장하는 표준 방식
- 결제창 SDK + 백엔드 승인 API 조합
